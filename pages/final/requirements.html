Final Project

For the final project, you will choose your own dataset to visualize using at least one advanced visualization technique. The guidelines can be summarized as follows:

Make something awesome.

The guidelines can be broken down as follows:

  • "Make" means you, individually, should come up with customized bespoke visualizations of your dataset.

  • "Something" refers to at least one advanced visualization using D3 version 5. Techniques include multivariate data visualizations (e.g. parallel coordinates, scatterplot matrices, or heatmaps), geospatial data visualizations (e.g. symbol maps, choropleths, or cartograms), hierarchical data visualizations (e.g. node-link trees, dendrograms, icicles, sunbursts, treemaps, or circle packing), or graph data visualizations (e.g. matrix diagrams, force-directed graphs, arc diagrams, edge bundling, or hive plots).

  • "Awesome" refers to visualizations that are highly interactive (e.g. mix of details on demand, linked views, brushing, filtering, or sorting), have strong data encodings, lots of context, high data-ink ratio, high data density, low lie factor, and are well-designed and aesthetically pleasing.

Each release of your final project will add new sections to your final project website. You must also participate in feedback exercises, create a 5 minute demonstration video, and participate in a video showing during finals week.

1. Project Dataset

The first step is to identify a dataset for your final project. You will need to produce somewhere between 3 to 4 distinct, interactive, complex visualizations of this dataset. 

1.1 Dataset Requirements

You need to identify a dataset that is "just right" in size and complexity. It should have a mix of different column types (continuous/discrete numerical, ordered/unordered categorical, date/time, latitude/longitude, etc.) and many rows (1k or more).

It is better to err on the larger side. If the dataset is too large, you can filter it down to a manageable size. If you find a dataset that is too small or simple, see if you can supplement it with a related (but comparable) dataset.

You can change your dataset later if this one ends up not working out, but you will lose a significant amount of time and effort if you switch.

2. Alpha Release

For this release of your final project, you need to setup your project website and create your first simple prototype.

2.1 Website Requirements

Each project release will add additional sections to your project website. For the alpha release, your website must contain the following sections:

  • Home/Landing Page: This should be a dedicated page that describes the overall narrative and provides clear navigation to the other pages. The overall narrative, goal, story, or theme includes what you hoped to learn by visualizing this dataset. Each visualization might have its own individual goal, but it must be relevant to the overall narrative stated here.

  • About Page: This should be a separate page that describes you and your motivation behind this project. This includes: your name, profile photo, biography, (optionally) professional social media accounts, and a paragraph explaining why you chose this dataset.

    For example, is the topic a passion of yours? Does it have personal significance? Were you inspired by something? This is different from the overall narrative, which often describes what you are trying to learn from this dataset instead of why you chose this dataset in the first place.

  • Dataset Page: This should be a separate page that describes the original dataset. It should include the dataset attribution (including link to the source and date accessed), the license for the dataset, the size of the dataset (including number of rows, columns, file size, or estimate where appropriate), and the content of the dataset listing all relevant columns and their descriptions.

  • Prototypes Page or Section: This should be a separate page (or pages) that will include your various prototypes and feedback you received on those prototypes. You will have to create at least 2 prototypes, but only one prototype is required for this specific project release.

You can update this content as you progress on this project, but should never delete a prototype. It captures the early attempts at visualizing the data, even if it is far from the final versions.

2.2 Prototype Requirements

Create a simple prototype of your dataset. It does not need to use an advanced technique. The goal is to make sure you are able to load your data and begin the exploration process. You want a prototype that will help you understand the dataset---leading to new visualization ideas for future releases.

You can use any tool to create this prototype (e.g. Tableau, Vega-Lite, RAW Graphs). The prototype can be static or interactive.

New Try to keep the prototype within 960 pixels wide (unless horizontal scrolling makes sense for your visualization) and 600 pixels tall (unless vertical scrolling makes sense for your visualization).
